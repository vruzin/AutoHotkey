# Plan_009 — Лаунчер 2.0 (IntelliJ "Search Everywhere" стиль)

## Цель
Превратить лаунчер в продвинутый поиск-всего: вкладки, аббревиатуры как данные,
редактирование хоткеев, горячие сокращения, поиск без учёта раскладки, богатый UI.

Решения (из брейншторма):
- Работа ПО ФАЗАМ с проверкой после каждой.
- Горячие сокращения (dp→docker ps) — только В ЛАУНЧЕРЕ (точное совпадение триггера).
- Аббревиатуры — мигрировать ВСЕ в data/abbreviations.json, динамическая регистрация.
- Редактирование хоткеев — захват комбинации (как в IDE).
- Поиск без раскладки — искать по обеим раскладкам сразу.
- Оверрайды хоткеев/триггеры — в settings.json (features.overrides).

## Архитектура
- core/Abbreviations.ahk — новый модуль (данные + динамическая регистрация).
- core/Commands.ahk — расширить Snapshot: scope, toggleable, enabled, trigger,
      abbr-команды; новые actions setHotkey/setTrigger/abbrAdd/abbrEdit/abbrDelete/abbrToggle.
- core/FeatureRegistry.ahk — overrides (key/trigger), SetKey.
- core/Settings.ahk — секция features.overrides в Defaults.
- features/HotkeyHuman.ahk — без изменений (уже есть).
- features/LayoutSearch — функция ru↔en раскладочной транслитерации для поиска
      (можно переиспользовать core/Layout.ahk Convert).
- ui/apps/settings/{app.js,style.css,index.html} — основная переработка UI.
- ui/Settings.ahk — новые обработчики моста.

## Ловушки/риски
- Hotstring() динамический: AHK v2 поддерживает Hotstring(":opts:abbr", callback).
      Старые :*: и :o: флаги перенести в options.
- Захват комбинации в WebView2: ловим keydown в JS, собираем модификаторы+клавишу,
      шлём в AHK как строку, конвертируем в AHK-синтаксис.
- Alt+буквы внутри лаунчера могут конфликтовать с системным Alt — preventDefault.
- Раскладочный поиск: использовать PuntoLayout.Convert (готово) для генерации
      второго варианта запроса.
- CapsLock-загадка (1/2/3 нажатия) — отдельно задокументировать в CapsLock_double,
      это вне лаунчера (вопрос пользователя информационный).
