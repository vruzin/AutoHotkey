# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Общее

Личная коллекция скриптов **AutoHotkey v1** (синтаксис legacy — `CapsLock & key::`, `Send,`, `IfWinActive` без скобок, директивы `#Include`, метки-подпрограммы через `Gosub`/`Return`). НЕ использовать синтаксис AHK v2 — все существующие модули несовместимы.

Точка входа — `main.ahk`. Запускается через ярлык `main — ярлык.lnk` (передаётся в установленный `AutoHotkey.exe` v1). Перезагрузка работающего скрипта — `CapsLock + R` (см. начало `main.ahk`).

## Архитектура

`main.ahk` — корневой модуль: глобальные хоткеи `CapsLock + …`, утилиты буфера обмена (`Send2`, `getSelText`), смена раскладки (`SetDefaultKeyboard`), переключатели темы/Shell в реестре. В конце файла через `#Include` подключаются остальные модули:

| Модуль | Назначение |
|---|---|
| `abbreviations.ahk` | Текстовые автозамены (`::ДДД::`, `:o:`, `:*:` — hotstrings AHK v1). Подставляются через `Send2`. |
| `GoogleTranslate.ahk` | `CapsLock + T` — перевод выделенного текста через Google (с `Shift` — ru→en, иначе auto→ru). Содержит `RemoveToolTip:` — общая метка скрытия `ToolTip`. |
| `kitty.ahk` | `CapsLock + K` — запуск KiTTY; `CapsLock + S` — динамически собранное меню `systemctl` команд (распарсивается строка-схема `Systemctl_services` с разделителями `\|` и `^`). |
| `main-menu.ahk` | `CapsLock + A` — главное меню (переключение темы Windows через реестр `Personalize\AppsUseLightTheme`). Здесь же иконка трея. |
| `dop_menu.ahk` | `CapsLock + Z` — служебные сниппеты (git config, пути). |
| `Eval.ahk` | Библиотека Pulover'а — вычисление выражений в строках (используется хоткеями калькуляции). |
| `build.ahk` | `CapsLock + B` — запуск `npm run dev` через Cmder для конкретных проектов (определяется по заголовку активного окна). |
| `Direct.ahk` | `CapsLock + W` — макрос Find&Replace для Yandex Direct Commander. |
| `fl.ahk` | `CapsLock + F` — шаблоны откликов для FL.ru. Использует локальную `Send3` (аналог `Send2` с другой задержкой). |
| `Docker.ahk` | `CapsLock + D` — иерархическое меню docker/docker-compose/swarm команд. |

Отдельно лежат, **не подключаются** через `main.ahk` и являются независимыми/архивными:
- `Punto.ahk`, `Punto2.ahk` — самостоятельные скрипты-замены Punto Switcher (класс `Punto`, ловят ввод через `Input`, по `Pause` исправляют раскладку).
- `CapsLock.ahk` — экспериментальная либа «тройного действия» одной клавиши (`sbTrioProcessingKeys`).
- `CapsLock_double.ahk` — простой 1/2/3/4-кратный обработчик `CapsLock` (одно нажатие — Ctrl+Shift для переключения языка, два — toggle CapsLock).
- `main1.ahk`, `Old.ahk` — архив прежних версий.

### Соглашения и общие утилиты

- **Ввод текста только через буфер обмена.** Прямой `Send` не работает корректно с русским / Unicode в большинстве приложений, поэтому везде используется `Send2(sText)` из `main.ahk`: сохраняет `ClipboardAll`, кладёт текст, `Send ^v`, восстанавливает. Любые новые автозамены/сниппеты должны идти через `Send2` (или локальный `Send3` в `fl.ahk` с меньшей задержкой). Не вызывать `SendRaw` для длинных строк.
- **Сброс залипших состояний.** Каждый хоткей, который выполняется по нажатию `CapsLock + …`, в конце вызывает `SetNumLockState, Off` и `SetCapsLockState, Off` — иначе `CapsLock` остаётся «включённым» после отпускания. Новые хоткеи должны делать то же самое.
- **Контекстные хоткеи.** Привязки к конкретным программам оборачиваются в `#IfWinActive ahk_exe …` / `#IfWinActive` (без условия — снимает контекст). Примеры: `Direct Commander.exe`, `GrphCalc.exe`, `idea64.exe` в `main.ahk`.
- **Меню.** Шаблон: `Menu, name, Add, &N. …, label_or_func` → `Menu, name, Show` → `Menu, name, DeleteAll`. `DeleteAll` после `Show` обязателен, иначе при повторном вызове пункты дублируются. Динамические меню (`kitty.ahk`, `Docker.ahk`) строятся через `Func("…").Bind(...)`.
- **Выделенный текст.** Получать только через `getSelText()` (определена в `main.ahk`) — она сохраняет/восстанавливает буфер обмена и обрабатывает таймаут `ClipWait`.

## Запуск, отладка, проверка

- В репозитории нет билд-системы, тестов и линтеров — это скрипт-конфиг, не приложение.
- Запуск: ярлык `main — ярлык.lnk` или `AutoHotkey.exe main.ahk` (требуется AHK v1.1.x).
- Горячая перезагрузка после правки: `CapsLock + R` (определено в начале `main.ahk`). При синтаксической ошибке AHK откроет окно с указанием строки.
- Иконка в трее — лампочка (`shell32.dll`, индекс 175, задаётся в `main-menu.ahk`). Через ПКМ → Reload This Script тоже можно перезагрузить.
- Внешние зависимости (пути захардкожены под текущую машину): `M:\Sys\Vivaldi\Application\vivaldi.exe`, `M:\Sys\GraphCalc\GrphCalc.exe`, `M:\Sys\OBS`, `M:\Sys\ZoomIt`, `M:\Sys\Kitty`, `M:\Sys\cmber\Cmder.exe`, `C:\Program Files\ABBYY FineReader 16\screenshotreader.exe`. При переносе на другую машину их нужно корректировать в `main.ahk`.

## Не-AHK файлы в репозитории

`graphcalc.ini`, `default.gcl`, `answer_history.gcs` — конфигурация и история внешнего калькулятора **GraphCalc** (запускается по `Win + C` из `main.ahk`). Эти файлы изменяются самим GraphCalc'ом; их попадание в `git status` после использования калькулятора — норма. Не редактировать вручную без причины.

## Стиль правок

- Сохранять синтаксис AHK v1: `;` — комментарий, метки заканчиваются `:` и завершаются `Return`, переменные через `%var%` или `% expr`. Не вводить v2-конструкции (`=>`, фигурные функции, `MsgBox(...)`).
- Комментарии и UI-строки — на русском (как в существующем коде).
- Новые модули подключать через `#Include имя.ahk` в самом конце `main.ahk`, рядом с остальными `#Include`.
- Хоткеи с `CapsLock` как префиксом — основной паттерн. Прямые `#`/`^`/`!`/`+` использовать только если действительно нужен глобальный шорткат вне «лаунчера CapsLock».
